SOUL.md is the "soul file" of an OpenClaw Agent—it defines the Agent's personality, behavioral guidelines, and system prompt. Precisely because SOUL.md sits at the top of the trust chain, tampering with it has the most far-reaching security implications. OpenClaw includes a built-in mechanism called `soul-evil` that serves both as a security research tool and as a fun "Agent personality test" feature.

## 24.5.1 Malicious SOUL Detection (`src/hooks/soul-evil.ts`)

### Mechanism Overview

`soul-evil` is a built-in hook that can replace the content of SOUL.md with another file—`SOUL_EVIL.md`—under specific conditions. This mechanism has two design intentions:

1. **Security Testing**: Let developers experience the effect of "SOUL being hijacked," thereby better understanding the importance of system prompt security
2. **Fun Feature**: Periodically or randomly switch Agent personality (e.g., make the Agent "mischievous" during specific time periods)

### Trigger Decision

The `decideSoulEvil` function determines whether to use the malicious SOUL based on two conditions:

```typescript
// src/hooks/soul-evil.ts (simplified)

export function decideSoulEvil(params: SoulEvilCheckParams): SoulEvilDecision {
  const fileName = evil?.file?.trim() || "SOUL_EVIL.md";
  if (!evil) return { useEvil: false, fileName };

  // Condition 1: Daily purge window
  const timeZone = resolveUserTimezone(params.userTimezone);
  const inPurge = isWithinDailyPurgeWindow({
    at: evil.purge?.at,
    duration: evil.purge?.duration,
    now: params.now ?? new Date(),
    timeZone,
  });
  if (inPurge) {
    return { useEvil: true, reason: "purge", fileName };
  }

  // Condition 2: Random probability
  const chance = clampChance(evil.chance);  // Clamp to [0, 1]
  if (chance > 0) {
    const random = params.random ?? Math.random;
    if (random() < chance) {
      return { useEvil: true, reason: "chance", fileName };
    }
  }

  return { useEvil: false, fileName };
}
```

### Daily Purge Window

The "purge window" allows configuring a fixed daily time period during which SOUL_EVIL.md is used. Time calculation takes user timezone into account:

```typescript
// src/hooks/soul-evil.ts (simplified)

function isWithinDailyPurgeWindow(params: {
  at?: string;          // e.g., "14:00"
  duration?: string;    // e.g., "30m"
  now: Date;
  timeZone: string;     // e.g., "Asia/Shanghai"
}): boolean {
  const startMinutes = parsePurgeAt(params.at);  // "14:00" → 840
  const durationMs = parseDurationMs(params.duration);

  // Convert current time to milliseconds offset in user timezone
  const nowMs = timeOfDayMsInTimezone(params.now, params.timeZone);

  const startMs = startMinutes * 60 * 1000;
  const endMs = startMs + durationMs;

  // Handle crossing midnight
  if (endMs < 24 * 60 * 60 * 1000) {
    return nowMs >= startMs && nowMs < endMs;
  }
  const wrappedEnd = endMs % (24 * 60 * 60 * 1000);
  return nowMs >= startMs || nowMs < wrappedEnd;
}
```

> **Sidebar**: `Intl.DateTimeFormat` is part of the ECMAScript Internationalization API (ECMA-402), which can format dates as representations in specific timezones. OpenClaw uses the `formatToParts` method to extract hours, minutes, and seconds in the user's timezone, avoiding manual handling of complex timezone offset calculations (such as daylight saving time transitions).

### SOUL Replacement Execution

`applySoulEvilOverride` performs the actual SOUL file replacement. It's called during the Agent bootstrap phase:

```typescript
// src/hooks/soul-evil.ts (simplified)

export async function applySoulEvilOverride(params: {
  files: WorkspaceBootstrapFile[];
  workspaceDir: string;
  config?: SoulEvilConfig;
}): Promise<WorkspaceBootstrapFile[]> {
  const decision = decideSoulEvil({ config: params.config, ... });
  if (!decision.useEvil) return params.files;

  // Read SOUL_EVIL.md content
  const evilPath = path.join(workspaceDir, decision.fileName);
  let evilContent: string;
  try {
    evilContent = await fs.readFile(evilPath, "utf-8");
  } catch {
    log.warn(`SOUL_EVIL active but file missing: ${evilPath}`);
    return params.files;  // File doesn't exist → fail safe
  }

  if (!evilContent.trim()) {
    log.warn(`SOUL_EVIL active but file empty: ${evilPath}`);
    return params.files;  // File is empty → fail safe
  }

  // Confirm SOUL.md exists in bootstrap files
  const hasSoulEntry = params.files.some((f) => f.name === "SOUL.md");
  if (!hasSoulEntry) {
    log.warn("SOUL_EVIL active but SOUL.md not in bootstrap files");
    return params.files;
  }

  // Replace SOUL.md content with SOUL_EVIL.md content
  return params.files.map((file) => {
    if (file.name !== "SOUL.md") return file;
    return { ...file, content: evilContent, missing: false };
  });
}
```

### Fail-Safe Design

Notice the multiple security checks in the function—if SOUL_EVIL.md doesn't exist, is empty, or SOUL.md is not in the bootstrap file list, the function silently falls back to normal behavior (returns the original file list). This "fail-safe" design ensures that even if configuration is incorrect, it won't break the Agent's normal startup.

export function resolveSoulEvilConfigFromHook(

  log?: SoulEvilLog,

  if (!entry) return null;

### Security Implications

Although `soul-evil` itself is a "fun" feature, it reveals the importance of SOUL.md security:

1. **SOUL.md is the Root of Trust**: All Agent behavior is ultimately constrained by SOUL.md. If SOUL.md is tampered with, the Agent's behavior will completely spiral out of control—and might even be weaponized.

2. **Workspace Security**: SOUL.md is stored in the workspace directory. Ensuring workspace directory file permissions (0o700) can prevent other users from modifying it.

3. **Monitoring and Auditing**: Triggering `soul-evil` logs debug messages. In production environments, monitoring these logs can detect anomalous SOUL replacement behavior.

4. **Least Privilege**: `soul-evil` can only replace SOUL.md's content, not modify other bootstrap files—this is a manifestation of the principle of least privilege.

## Summary of This Section

1. **`soul-evil`** is a built-in hook that can replace SOUL.md with SOUL_EVIL.md under specific conditions, used for security testing or fun personality switching.
2. **Two trigger conditions**: Daily purge window (fixed time period) and random probability (floating point between 0-1).
3. **Timezone-aware**: Purge window uses `Intl.DateTimeFormat` for time calculation in user timezone, supporting midnight-crossing scenarios.
4. **Fail-safe design**: When file doesn't exist, is empty, or SOUL.md is missing, silently falls back to normal behavior.
5. **Root of Trust Protection**: SOUL.md sits at the top of the trust chain; its security depends on workspace directory permissions and filesystem auditing.
