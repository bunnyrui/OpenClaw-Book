OpenClaw provides a fully-featured Terminal User Interface (TUI) that allows users to interact with AI Agents directly in the terminal through interactive conversations. The TUI is not merely a simple command-line REPL—it features syntax highlighting, Markdown rendering, tool call visualization, multi-Agent switching, session management, and other complete functionality.

> **Sidebar: The Difference Between TUI and CLI**
> CLI (Command Line Interface) is the traditional command-line tool—users input commands, the program returns results, and interaction is linear. TUI (Terminal User Interface), on the other hand, constructs an interface similar to a GUI within the terminal, featuring layout systems, interactive components, and real-time updates. Classic TUI applications include `vim`, `htop`, and `tmux`. TUI controls terminal cursor position, text color, and screen regions through ANSI escape codes.

### TUI Component Tree

The TUI interface is composed of a component tree:

```
TUI (root)
└── root (Container)
    ├── header (Text)         ← Title bar: URL / Agent / Session
    ├── chatLog (ChatLog)     ← Chat history area
    ├── statusContainer       ← Status line (loading animation / messages)
    ├── footer (Text)         ← Bottom info bar
    └── editor (CustomEditor) ← Input editor
```

```typescript
// src/tui/tui.ts (simplified)

export async function runTui(opts: TuiOptions) {
  const tui = new TUI(new ProcessTerminal());
  const header = new Text("", 1, 0);
  const statusContainer = new Container();
  const footer = new Text("", 1, 0);
  const chatLog = new ChatLog();
  const editor = new CustomEditor(tui, editorTheme);

  const root = new Container();
  root.addChild(header);
  root.addChild(chatLog);
  root.addChild(statusContainer);
  root.addChild(footer);
  root.addChild(editor);

  tui.addChild(root);
  tui.setFocus(editor);  // Focus defaults to editor
}
```

`ProcessTerminal` is the low-level terminal adapter, encapsulating `process.stdin` / `process.stdout` read/write and terminal size detection.

### Gateway Client

TUI communicates with Gateway through `GatewayChatClient`:

```typescript
// src/tui/gateway-chat.ts (simplified)

export class GatewayChatClient {
  private client: GatewayClient;

  constructor(opts: GatewayConnectionOptions) {
    this.client = new GatewayClient({
      url: resolveUrl(opts),
      token: opts.token,
      password: opts.password,
      clientName: GATEWAY_CLIENT_NAMES.TUI,
      mode: GATEWAY_CLIENT_MODES.TUI,
      onEvent: (evt) => { /* forward events */ },
      onHelloOk: () => { /* connection success callback */ },
      onClose: () => { /* disconnect callback */ },
    });
  }

  async sendMessage(opts: ChatSendOptions) {
    return this.client.request("chat.send", {
      sessionKey: opts.sessionKey,
      message: opts.message,
      thinking: opts.thinking,
      deliver: opts.deliver,
      idempotencyKey: opts.runId ?? randomUUID(),
    }, { expectFinal: true });
  }

  async listSessions(params) {
    return this.client.request("sessions.list", params);
  }

  async patchSession(params) {
    return this.client.request("sessions.patch", params);
  }
}
```

### Input Handling

The editor's submit handler supports three input modes:

```typescript
export function createEditorSubmitHandler(params) {
  return (text: string) => {
    const raw = text;
    const value = raw.trim();
    params.editor.setText("");

    if (!value) return;  // Ignore empty input

    // Mode 1: Bash command (starts with !)
    if (raw.startsWith("!") && raw !== "!") {
      params.editor.addToHistory(raw);
      void params.handleBangLine(raw);
      return;
    }

    params.editor.addToHistory(value);

    // Mode 2: Slash command (starts with /)
    if (value.startsWith("/")) {
      void params.handleCommand(value);
      return;
    }

    // Mode 3: Regular message
    void params.sendMessage(value);
  };
}
```

Three input modes:

| Prefix | Mode | Example | Behavior |
|--------|------|---------|----------|
| `!` | Bash | `!ls -la` | Execute command in local Shell |
| `/` | Command | `/think high` | Execute slash command |
| None | Message | `Write a function for me` | Send to AI Agent |

Note that `!` mode uses **raw untrimmed text** (`raw`), ensuring leading spaces don't accidentally trigger Bash mode. A standalone `!` is treated as a regular message.

### Streaming Response Assembly

`TuiStreamAssembler` is responsible for assembling Gateway's incremental events into complete display text:

```typescript
export class TuiStreamAssembler {
  private runs = new Map<string, RunStreamState>();

  // Process incremental updates
  ingestDelta(
    runId: string,
    message: unknown,
    showThinking: boolean,
  ): string | null {
    const state = this.getOrCreateRun(runId);
    const previousDisplayText = state.displayText;

    // Extract thinking process and content separately
    const thinkingText = extractThinkingFromMessage(message);
    const contentText = extractContentFromMessage(message);
    if (thinkingText) state.thinkingText = thinkingText;
    if (contentText) state.contentText = contentText;

    // Compose display text
    state.displayText = composeThinkingAndContent({
      thinkingText: state.thinkingText,
      contentText: state.contentText,
      showThinking,
    });

    // Only return if text actually changed
    if (state.displayText === previousDisplayText) return null;
    return state.displayText;
  }

  // Finalize: clean up run state
  finalize(runId: string, message: unknown, showThinking: boolean) {
    // ... final assembly + cleanup
    this.runs.delete(runId);
    return finalText;
  }
}
```

Each run (`runId`) maintains independent state: `thinkingText` (thinking process) and `contentText` (main content) are tracked separately and composed into display text as needed. The `showThinking` parameter controls whether to display the thinking process (such as Claude's `<thinking>` tag content) to the user.

## Section Summary

1. **pi-tui framework**: TUI is built on `@mariozechner/pi-tui`, using a component tree model (header → chatLog → status → footer → editor).
2. **Gateway communication**: `GatewayChatClient` wraps the WebSocket client, providing high-level APIs like `sendMessage`/`listSessions`/`patchSession`.
3. **Three input modes**: `!` prefix executes local Shell, `/` prefix triggers slash commands, no prefix sends message to Agent.
4. **Streaming response assembly**: `TuiStreamAssembler` separately tracks thinking process and main content, composing display text as needed, supporting incremental updates.
5. **Theme system**: Carefully designed 20+ color palette with syntax highlighting (`cli-highlight`), providing comfortable reading in dark terminals.
6. **Slash commands**: 20+ commands covering Agent/session/model management, supporting argument autocomplete and command aliases.
7. **Overlay selectors**: Fuzzy search lists for Agent/session/model selection, providing interaction experience similar to VS Code's command palette.
8. **Terminal utilities layer**: `src/terminal/` provides low-level capabilities like ANSI control, table rendering, stream writing, and hyperlinks.
