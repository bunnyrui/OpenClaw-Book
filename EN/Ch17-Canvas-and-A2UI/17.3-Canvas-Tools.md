## 17.3.1 push / reset / eval / snapshot Operations

### Tool Registration and Schema

Canvas tool defines 7 actions (action), unified in a flat TypeBox Schema:

```typescript
// src/agents/tools/canvas-tool.ts

const CANVAS_ACTIONS = [
  "present",     // Show Canvas
  "hide",        // Hide Canvas
  "navigate",    // Navigate to specified URL
  "eval",        // Execute JavaScript in Canvas
  "snapshot",    // Capture Canvas snapshot
  "a2ui_push",   // Push A2UI JSONL data
  "a2ui_reset",  // Reset A2UI render state
] as const;

const CanvasToolSchema = Type.Object({
  action: stringEnum(CANVAS_ACTIONS),       // Required: action type
  node: Type.Optional(Type.String()),       // Target node
  // present-specific
  target: Type.Optional(Type.String()),     // URL or path
  x: Type.Optional(Type.Number()),          // Placement coordinates
  y: Type.Optional(Type.Number()),
  width: Type.Optional(Type.Number()),      // Placement dimensions
  height: Type.Optional(Type.Number()),
  // navigate-specific
  url: Type.Optional(Type.String()),
  // eval-specific
  javaScript: Type.Optional(Type.String()),
  // snapshot-specific
  outputFormat: optionalStringEnum(["png", "jpg", "jpeg"]),
  maxWidth: Type.Optional(Type.Number()),
  quality: Type.Optional(Type.Number()),
  delayMs: Type.Optional(Type.Number()),
  // a2ui_push-specific
  jsonl: Type.Optional(Type.String()),      // Inline JSONL
  jsonlPath: Type.Optional(Type.String()),  // JSONL file path
});
```

This is a **flat design**—all action parameters mixed in one Schema, runtime validates required parameters based on `action` field. Benefit: LLM only learns one tool's parameter structure, not 7 separate tools.

### Gateway Proxy Pattern

All Canvas tool operations don't communicate directly with Canvas Host, but forward through Gateway's `node.invoke`:

```typescript
// src/agents/tools/canvas-tool.ts — invoke helper function

const nodeId = await resolveNodeId(gatewayOpts, params.node, true);

const invoke = async (command: string, invokeParams?: Record<string, unknown>) =>
  await callGatewayTool("node.invoke", gatewayOpts, {
    nodeId,                            // Target node ID
    command,                           // e.g., "canvas.present"
    params: invokeParams,              // Command parameters
    idempotencyKey: crypto.randomUUID(), // Idempotency key
  });
```

Data flow:

```
Agent calls canvas tool
    │
    ↓ callGatewayTool("node.invoke", ...)
Gateway Protocol
    │
    ↓ node.invoke RPC
Native App Node (iOS/Android/macOS)
    │
    ↓ Node Command Executor
Execute operation in WebView
```

Each call includes an `idempotencyKey` (idempotency key), ensuring network retransmissions don't cause duplicate operations.

### Action-by-Action Analysis

#### present: Show Canvas

```typescript
case "present": {
  const placement = {
    x: typeof params.x === "number" ? params.x : undefined,
    y: typeof params.y === "number" ? params.y : undefined,
    width: typeof params.width === "number" ? params.width : undefined,
    height: typeof params.height === "number" ? params.height : undefined,
  };
  const invokeParams: Record<string, unknown> = {};
  if (typeof params.target === "string" && params.target.trim()) {
    invokeParams.url = params.target.trim();
  }
  if (Number.isFinite(placement.x) || ...) {
    invokeParams.placement = placement;
  }
  await invoke("canvas.present", invokeParams);
  return jsonResult({ ok: true });
}
```

`present` can specify:
- **target**—URL to load (usually Canvas Host path like `http://localhost:PORT/__openclaw__/canvas/`)
- **placement**—Canvas window position and size on screen (`x`/`y`/`width`/`height`), supports precise positioning

#### hide: Hide Canvas

```typescript
case "hide":
  await invoke("canvas.hide", undefined);
  return jsonResult({ ok: true });
```

Simplest operation—no parameters, just hide.

#### navigate: Navigate to URL

```typescript
case "navigate": {
  const url = readStringParam(params, "url", { required: true });
  await invoke("canvas.navigate", { url });
  return jsonResult({ ok: true });
}
```

Navigate to new URL in already-displayed Canvas WebView, no need to hide then present.

#### eval: Execute JavaScript

```typescript
case "eval": {
  const javaScript = readStringParam(params, "javaScript", { required: true });
  const raw = await invoke("canvas.eval", { javaScript });
  const result = raw?.payload?.result;
  if (result) {
    return { content: [{ type: "text", text: result }], details: { result } };
  }
  return jsonResult({ ok: true });
}
```

`eval` executes arbitrary JavaScript code in Canvas WebView and returns execution result. This is Canvas's most flexible operation—Agents can use it to dynamically modify pages, read DOM state, trigger events, etc. If return value exists, it's returned as text content to Agent.

#### snapshot: Capture Snapshot

```typescript
case "snapshot": {
  const format = formatRaw === "jpg" || formatRaw === "jpeg" ? "jpeg" : "png";
  const raw = await invoke("canvas.snapshot", { format, maxWidth, quality });
  
  // Parse snapshot payload
  const payload = parseCanvasSnapshotPayload(raw?.payload);
  
  // Save as temporary file
  const filePath = canvasSnapshotTempPath({
    ext: payload.format === "jpeg" ? "jpg" : payload.format,
  });
  await writeBase64ToFile(filePath, payload.base64);
  
  // Return image result (base64 + file path)
  return await imageResult({
    label: "canvas:snapshot",
    path: filePath,
    base64: payload.base64,
    mimeType: imageMimeFromFormat(payload.format) ?? "image/png",
  });
}
```

`snapshot` is the most complex Canvas tool operation:

1. Send `canvas.snapshot` command to node, node screenshots WebView
2. Node returns base64-encoded image data
3. `parseCanvasSnapshotPayload()` parses payload (validates `format` and `base64` fields)
4. `writeBase64ToFile()` writes base64 to temporary file (`/tmp/openclaw-canvas-snapshot-{uuid}.{ext}`)
5. Return as `imageResult` format, Agent can "see" screenshot content

Snapshot temporary file path format:

```typescript
// src/cli/nodes-canvas.ts
export function canvasSnapshotTempPath(opts: { ext: string }) {
  const tmpDir = opts.tmpDir ?? os.tmpdir();
  const id = opts.id ?? randomUUID();
  const ext = opts.ext.startsWith(".") ? opts.ext : `.${opts.ext}`;
  return path.join(tmpDir, `${cliName}-canvas-snapshot-${id}${ext}`);
  // Example: /tmp/openclaw-canvas-snapshot-a1b2c3d4.jpg
}
```

#### a2ui_push: Push A2UI Data

```typescript
case "a2ui_push": {
  // Support both inline JSONL or file path
  const jsonl =
    typeof params.jsonl === "string" && params.jsonl.trim()
      ? params.jsonl                                          // Inline
      : typeof params.jsonlPath === "string"
        ? await fs.readFile(params.jsonlPath.trim(), "utf8")  // Read from file
        : "";
  if (!jsonl.trim()) {
    throw new Error("jsonl or jsonlPath required");
  }
  await invoke("canvas.a2ui.pushJSONL", { jsonl });
  return jsonResult({ ok: true });
}
```

`a2ui_push` supports two input methods:
- **`jsonl`**—Provide JSONL string directly in parameters (for small UI instructions)
- **`jsonlPath`**—Specify JSONL file path (for large UI instructions, Agent writes file first then pushes)

#### a2ui_reset: Reset A2UI State

```typescript
case "a2ui_reset":
  await invoke("canvas.a2ui.reset", undefined);
  return jsonResult({ ok: true });
```

Clear all A2UI renderer state, restore to initial blank state.

## 17.3.2 Canvas Surface on macOS / iOS / Android

### Node Command Policy

Canvas command availability across platforms is controlled by **node command policy** in `node-command-policy.ts`:

```typescript
// src/gateway/node-command-policy.ts

const CANVAS_COMMANDS = [
  "canvas.present", "canvas.hide", "canvas.navigate",
  "canvas.eval", "canvas.snapshot",
  "canvas.a2ui.push", "canvas.a2ui.pushJSONL", "canvas.a2ui.reset",
];

const PLATFORM_DEFAULTS: Record<string, string[]> = {
  ios:     [...CANVAS_COMMANDS, ...CAMERA, ...SCREEN, ...LOCATION],
  android: [...CANVAS_COMMANDS, ...CAMERA, ...SCREEN, ...LOCATION, ...SMS],
  macos:   [...CANVAS_COMMANDS, ...CAMERA, ...SCREEN, ...LOCATION, ...SYSTEM],
  linux:   [...SYSTEM_COMMANDS],       // No Canvas
  windows: [...SYSTEM_COMMANDS],       // No Canvas
};
```

Canvas is available on **iOS, Android, macOS**, unavailable on **Linux and Windows**. This is because Canvas depends on native app WebView—Linux and Windows OpenClaw runs in headless mode without graphical interface.

### Command Access Control

Node command policy implements three-layer access control:

```typescript
// src/gateway/node-command-policy.ts

export function resolveNodeCommandAllowlist(cfg, node): Set<string> {
  // 1. Platform default allowlist
  const platformId = normalizePlatformId(node?.platform, node?.deviceFamily);
  const base = PLATFORM_DEFAULTS[platformId] ?? PLATFORM_DEFAULTS.unknown;

  // 2. Configuration file extensions
  const extra = cfg.gateway?.nodes?.allowCommands ?? [];

  // 3. Configuration file denials
  const deny = new Set(cfg.gateway?.nodes?.denyCommands ?? []);

  // Merge: (platform default ∪ config extension) - config denial
  const allow = new Set([...base, ...extra]);
  for (const blocked of deny) allow.delete(blocked);
  return allow;
}

export function isNodeCommandAllowed(params): { ok: true } | { ok: false; reason: string } {
  // Check 1: Command in allowlist
  if (!params.allowlist.has(command)) {
    return { ok: false, reason: "command not allowlisted" };
  }
  // Check 2: Node declared this command
  if (!params.declaredCommands?.includes(command)) {
    return { ok: false, reason: "command not declared by node" };
  }
  return { ok: true };
}
```

| Check Layer | Description | Example |
|---|---|---|
| Platform Allowlist | Default commands based on node platform | iOS defaults allow Canvas + Camera |
| Config Extension | `gateway.nodes.allowCommands` can add extra | Allow Canvas on Linux node |
| Config Denial | `gateway.nodes.denyCommands` can block | Block screenshot on Android node |
| Node Declaration | Node declares supported commands at registration | Node not declaring Canvas means unavailable |

### WebView Implementation Across Platforms

Canvas tool forwards commands to target nodes via Gateway, but **actual WebView rendering happens on native app side**. Different platform implementations:

| Platform | WebView Technology | Canvas Implementation | Features |
|---|---|---|---|
| **macOS** | `WKWebView` | `apps/macos/` (Swift) | Desktop window, placement positioning, system.run support |
| **iOS** | `WKWebView` | `apps/ios/` (Swift) | Full-screen or split-screen, safe area adaptation, camera/location support |
| **Android** | `android.webkit.WebView` | `apps/android/` (Kotlin) | Full-screen or split-screen, high-contrast theme, SMS support |

All three platforms' WebView need:
1. **Load Canvas Host URL**—Usually `http://<gateway-host>:<canvas-port>/__openclaw__/a2ui`
2. **Inject JS Bridge**—iOS via `WKScriptMessageHandler`, Android via `@JavascriptInterface`
3. **Handle node commands**—Receive `canvas.present`/`canvas.eval` etc. and execute in WebView

### Command-Line Interface

Besides Agent tools, Canvas operations can also be called directly via CLI:

```bash
# Capture snapshot
openclaw nodes canvas snapshot --node my-iphone --format jpg --max-width 1024

# Show Canvas
openclaw nodes canvas present --node my-iphone --target http://localhost:8080/

# Execute JS in Canvas
openclaw nodes canvas eval --node my-iphone --js "document.title"

# Push A2UI text
openclaw nodes canvas a2ui push --node my-iphone --text "Hello from CLI"

# Push A2UI JSONL file
openclaw nodes canvas a2ui push --node my-iphone --jsonl ./ui-commands.jsonl

# Reset A2UI
openclaw nodes canvas a2ui reset --node my-iphone
```

CLI implementation is in `src/cli/nodes-cli/register.canvas.ts`, sharing the same `invokeCanvas()` helper function with Agent tools, ultimately calling Gateway via `callGatewayCli("node.invoke", ...)`.

CLI's `a2ui push` command additionally supports `--text` parameter—quickly convert plain text to A2UI JSONL format without manually writing JSONL:

```typescript
// src/cli/nodes-cli/register.canvas.ts — a2ui push

const jsonl = hasText
  ? buildA2UITextJsonl(String(opts.text))  // Plain text → JSONL
  : await fs.readFile(String(opts.jsonl), "utf8");

const { version, messageCount } = validateA2UIJsonl(jsonl);
if (version === "v0.9") {
  throw new Error(
    "Detected A2UI v0.9 JSONL (createSurface). OpenClaw currently supports v0.8 only."
  );
}
```

Note version check: OpenClaw currently only supports A2UI v0.8 protocol, errors if detecting v0.9's `createSurface` instruction.

### End-to-End Workflow Example

A complete Canvas usage scenario—Agent generates data visualization for user:

```
1. Agent generates HTML file
   → Write to ~/.openclaw/canvas/chart.html

2. Agent calls canvas present
   → canvas-tool.ts → callGatewayTool("node.invoke")
   → Gateway → node.invoke → iOS app
   → iOS opens WebView, loads http://host:port/__openclaw__/canvas/chart.html
   → Canvas Host serves chart.html (injects Live Reload)
   → WebView renders chart

3. Agent calls canvas snapshot
   → Same path to iOS app
   → iOS captures WebView snapshot
   → Base64 encode return
   → Agent "sees" render result, confirms correct

4. User interacts with chart (clicks data point)
   → JS in WebView calls openclawSendUserAction()
   → iOS WKScriptMessageHandler receives
   → Message returns to Agent
   → Agent updates chart based on user action
```

## Section Summary

1. Canvas tool unifies 7 actions into **flat Schema**, LLM only learns one tool interface
2. All operations **proxy to target node** via `callGatewayTool("node.invoke")`, don't communicate directly with Canvas Host
3. `snapshot` is most complex operation, involving node screenshot → base64 encode → parse payload → write temp file → return image result
4. `a2ui_push` supports both inline JSONL and file path input, CLI additionally supports `--text` shortcut
5. Canvas available on **iOS / Android / macOS**, Linux / Windows unsupported due to lack of WebView
6. **Node command policy** implements four-layer access control: platform default → config extension → config denial → node declaration
7. CLI and Agent tools share same underlying call chain, both reach target node via Gateway's `node.invoke`
