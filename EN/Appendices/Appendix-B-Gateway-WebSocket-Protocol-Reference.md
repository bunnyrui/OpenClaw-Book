## Frame Format

All WebSocket messages use JSON format and contain three frame types:

```typescript
// Request frame (client → Gateway)
{ "type": "req", "id": "req-001", "method": "chat.send", "params": { ... } }

// Response frame (Gateway → client)
{ "type": "res", "id": "req-001", "result": { ... } }
// or error response
{ "type": "res", "id": "req-001", "error": "error message" }

// Event frame (Gateway → client, push)
{ "type": "event", "event": "chat", "payload": { ... }, "seq": 42 }
```

## Request Methods Quick Reference

### Chat

| Method | Parameters | Description |
|--------|-----------|-------------|
| `chat.send` | `sessionKey, message, channel?, deliver?, thinking?, idempotencyKey?` | Send message to Agent |
| `chat.abort` | `sessionKey` | Abort current Agent run |

### Session Management

| Method | Parameters | Description |
|--------|-----------|-------------|
| `sessions.list` | `limit?, agentId?` | List sessions |
| `sessions.patch` | `key, model?, thinking?, label?` | Modify session properties |
| `sessions.reset` | `key` | Reset session |
| `sessions.delete` | `key, deleteTranscript?` | Delete session |
| `sessions.resolve` | `key? \| label?` | Find session by key or label |
| `sessions.history` | `key, limit?` | Get session history |

### Agent

| Method | Parameters | Description |

| `agent.wait` | `runId, timeoutMs` | Wait for Agent run to complete |
| `agents.list` | — | List all Agents |

## Event Types

| Event Name | Trigger Condition | Payload Key Fields |
|------------|------------------|-------------------|
| `chat` | Chat state change | `sessionKey, state(delta/final/aborted/error), message` |
| `agent` | Agent activity | `sessionKey, stream(tool/lifecycle), data` |
| `presence` | Online status change | `host, version, mode` |
| `config` | Configuration change | `key, value` |
| `session` | Session change | `key, action(created/updated/deleted)` |

## Connection Handshake

After connecting, the client sends a `hello` request:

```json
{
  "type": "req",
  "id": "hello-1",
  "method": "hello",
  "params": {
    "protocolVersion": 1,
    "clientName": "webchat",
    "clientVersion": "1.0.0",
    "token": "auth-token",
    "password": "auth-password",
    "capabilities": ["chat", "events"]
  }
}
```

Gateway responds:

```json
{
  "type": "res",
  "id": "hello-1",
  "result": {
    "ok": true,
    "protocolVersion": 1,
    "serverVersion": "2026.2.1",
    "features": ["chat", "sessions", "agents", "tools"]
  }
}
```
