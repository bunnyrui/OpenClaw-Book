# Appendix C: Tool Definition Reference

> **Generated by**: Claude Opus 4.6 (anthropic/claude-opus-4-6)

All built-in tool definitions in OpenClaw. Each tool is defined through JSON Schema for input parameters.

## File System Tools

| Tool Name | Description | Key Parameters |
|-----------|-------------|-----------------|
| `read` | Read file contents | `filePath, offset?, limit?` |
| `write` | Write to file | `filePath, content` |
| `edit` | Edit file (find and replace) | `filePath, oldString, newString, replaceAll?` |
| `apply_patch` | Apply unified diff patch | `filePath, patch` |

## Shell Tools

| Tool Name | Description | Key Parameters |
|-----------|-------------|-----------------|
| `exec` | Execute shell command | `command, workdir?, timeout?` |
| `process` | Process management (list/kill) | `action, pid?` |

## Browser Tools

| Tool Name | Description | Key Parameters |
|-----------|-------------|-----------------|
| `browser` | Browser automation | `action(navigate/click/type/screenshot/...), url?, selector?, text?` |

## Session Tools

| Tool Name | Description | Key Parameters |
|-----------|-------------|-----------------|
| `sessions_spawn` | Spawn sub-agent | `task, agentId?, model?, label?, cleanup?` |
| `sessions_list` | List sessions | `limit?` |
| `sessions_history` | View session history | `sessionKey, limit?` |
| `sessions_send` | Send message to session | `sessionKey, message` |
| `session_status` | View session status | `sessionKey?` |

## Memory Tools

| Tool Name | Description | Key Parameters |

## Sandbox Default Policy

```
Allow: exec, process, read, write, edit, apply_patch, image,
       sessions_list, sessions_history, sessions_send,
       sessions_spawn, session_status

Deny: browser, canvas, nodes, cron, gateway,
      telegram, discord, slack, whatsapp, signal, ...
```
