The Android application, like the iOS application, plays the role of a "node"—connecting to a Gateway and exposing the device's hardware capabilities. It is built with **Kotlin + Jetpack Compose** and is the only native client running on a non-Apple platform among the three native clients.

## 28.3.1 Kotlin Architecture

### Project Configuration

The Android application is located in `apps/android/` and uses Gradle for building:

```kotlin
// apps/android/app/build.gradle.kts
android {
    namespace = "ai.openclaw.android"
    compileSdk = 36
    defaultConfig {
        applicationId = "ai.openclaw.android"
        minSdk = 31        // Android 12 (S) minimum
        targetSdk = 36
    }
    buildFeatures {
        compose = true      // Jetpack Compose UI
        buildConfig = true
    }
}
```

It's worth noting that `assets.srcDir` points to the shared library's resource directory—the Android app directly reuses resource files from `OpenClawKit` (such as sound effects and icons), but since Kotlin cannot directly use Swift packages, protocols and models need to be reimplemented in Kotlin.

### NodeRuntime Core

`NodeRuntime` is the core runtime of the Android application, aggregating all device capabilities:

```kotlin
// apps/android/app/src/main/java/ai/openclaw/android/NodeRuntime.kt
class NodeRuntime(context: Context) {
    private val scope = CoroutineScope(SupervisorJob() + Dispatchers.IO)
    
    val prefs = SecurePrefs(appContext)
    val canvas = CanvasController()
    val camera = CameraCaptureManager(appContext)
    val location = LocationCaptureManager(appContext)
    val screenRecorder = ScreenRecordManager(appContext)
    val sms = SmsManager(appContext)
    
    private val voiceWake: VoiceWakeManager by lazy {
        VoiceWakeManager(context = appContext, scope = scope,
            onCommand = { command -> ... })
    }
}
```

Corresponding to iOS's `NodeAppModel`, `NodeRuntime` creates and holds instances of all capability managers. It uses Kotlin coroutines (`CoroutineScope` + `SupervisorJob`) for asynchronous operations.

> **Sidebar — Kotlin Coroutines and SupervisorJob**
>
> Kotlin coroutines are Kotlin's asynchronous programming solution, similar to Swift's async/await. `CoroutineScope` defines the lifecycle scope of coroutines, and `SupervisorJob` ensures that the failure of one child coroutine does not cancel other sibling coroutines—this is important for node runtime because camera failure should not affect location services.

### Foreground Service

Android requires long-running background tasks to be implemented through **Foreground Services**. `NodeForegroundService` is started in `MainActivity.onCreate`:

```kotlin
// MainActivity.kt
override fun onCreate(savedInstanceState: Bundle?) {
    NodeForegroundService.start(this)  // Start foreground service
    // ...
}
```

The foreground service displays a persistent notification in the notification bar, informing the user that OpenClaw is running. This ensures that the WebSocket connection is not killed by the system even if the user switches to other applications.

## 28.3.2 Canvas / Camera / Screen Recording

The device capabilities exposed by the Android node align with iOS:

| Capability | Manager | Android API |
|------------|---------|-------------|
| Canvas | `CanvasController` | WebView (optional) |
| Camera | `CameraCaptureManager` | Camera2 API |
| Screen Recording | `ScreenRecordManager` | MediaProjection API |
| Location | `LocationCaptureManager` | LocationManager |
| SMS | `SmsManager` | SmsManager API |

Each capability manager needs to integrate with Android's permission system. `PermissionRequester` handles runtime permission requests uniformly:

```kotlin
// MainActivity.kt
permissionRequester = PermissionRequester(this)
viewModel.camera.attachPermissionRequester(permissionRequester)
viewModel.sms.attachPermissionRequester(permissionRequester)
viewModel.screenRecorder.attachScreenCaptureRequester(screenCaptureRequester)
```

Screen recording is special: Android's `MediaProjection` API not only requires runtime permissions but also requires explicit user consent in a system dialog, so it has a separate `ScreenCaptureRequester`.

MainActivity also monitors the `preventSleep` state flow, setting `FLAG_KEEP_SCREEN_ON` when needed to prevent the screen from turning off automatically—this is important during Canvas display or conversation mode.

## 28.3.3 Optional SMS Support

An Android-exclusive capability is **SMS text messaging** support. When users grant SMS permissions, AI agents can read and send SMS messages through `SmsManager`. This is not possible on iOS due to system restrictions.

The declaration and use of SMS capability follows the same pattern as other capabilities: declared via capabilities when connecting to the Gateway, and the Gateway decides whether to expose SMS-related tools based on this.

```
OpenClawCapability:
  - canvas
  - camera
  - screen
  - voiceWake
  - location
  - sms          ← Android-exclusive
```
