## 34.1.1 Requirements Analysis

### Core Requirements

MiniClaw aims to implement the following features:

| Feature | Corresponding OpenClaw Module | MVP Scope |
|---------|-------------------------------|-----------|
| Gateway Server | `src/gateway/` | WebSocket server + RPC protocol |
| Session Management | `src/sessions/` | In-memory session storage + basic routing |
| AI Agent Runtime | `src/agents/` | Single Agent + tool invocation loop |
| Message Channels | `src/channels/` | Telegram Bot + WebChat |
| Tool System | `src/agents/tools/` | Bash execution + file read/write |
| Memory System | `src/memory/` | Markdown file memory |

### Non-Requirements (Excluded from MVP)

The following features are not implemented in the MVP but have reserved extension interfaces:

- Multi-agent architecture
- Sandbox isolation
- Canvas / A2UI
- Browser automation
- Cron scheduling
- Native clients
- Skill system

### User Stories

```
As a user, I want to:
1. Send messages to my AI assistant via Telegram
2. Interact with my AI assistant through a web interface
3. Have the AI assistant execute local commands (e.g., view files, run scripts)
4. Have the AI assistant remember things I've told it previously
5. Deploy the assistant on a VPS and use it anytime, anywhere
```

### LLM API Selection

MiniClaw should support at least two LLM providers:

1. **Anthropic Claude**: Via `@anthropic-ai/sdk`, supporting tool invocation, streaming output, thinking process
2. **OpenAI GPT**: Via `openai` SDK, supporting function calling, streaming output

It's recommended to use the **Adapter Pattern** (referencing OpenClaw's `src/providers/`) to unify API differences across providers:

```typescript
interface LLMProvider {
  chat(params: {
    messages: Message[];
    tools?: ToolDefinition[];
    stream?: boolean;
  }): AsyncIterable<ChatChunk>;
}

class AnthropicProvider implements LLMProvider { /* ... */ }
class OpenAIProvider implements LLMProvider { /* ... */ }
```

## Summary of This Section

1. **MiniClaw** is a minimal viable implementation of OpenClaw's core design, focusing on four major modules: Gateway + Agent + Channels + Tools.
2. **Architecture design** borrows from OpenClaw's layered pattern but removes advanced features like multi-agent, sandbox, and Canvas.
3. **Technology selection** recommends TypeScript + Node.js 22+, with dependencies on `ws` (WebSocket), `openai`/`@anthropic-ai/sdk` (LLM), and `grammy` (Telegram).
4. **Adapter pattern** unifies API differences across different LLM providers, facilitating future extensions.
5. **Four-phase development plan**: Gateway вҶ’ Channels вҶ’ Tools/Memory вҶ’ Deployment, totaling 22-32 hours.
