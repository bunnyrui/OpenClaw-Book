## 34.5.1 Docker Containerization

### Dockerfile

Referencing OpenClaw's multi-stage build (Chapter 29), write a Dockerfile for MiniClaw:

```dockerfile
# Dockerfile

# Stage 1: Build
FROM node:22-slim AS builder
WORKDIR /app
COPY package.json pnpm-lock.yaml ./
RUN corepack enable && pnpm install --frozen-lockfile
COPY . .
RUN pnpm build

# Stage 2: Runtime
FROM node:22-slim AS runtime
RUN groupadd -r miniclaw && useradd -r -g miniclaw -m miniclaw
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/package.json .
COPY --from=builder /app/ui ./ui

# Run as non-root user
USER miniclaw

# Data directory
VOLUME /home/miniclaw/.miniclaw
ENV NODE_ENV=production

EXPOSE 3000 8080
CMD ["node", "dist/index.js"]
```

Key security measures:

| Measure | Description |
|---------|-------------|
| Multi-stage build | Production image doesn't include build tools or source code |
| Non-root user | Run with `miniclaw` user, reducing privileges |
| `--frozen-lockfile` | Ensures dependency versions match development environment |
| VOLUME mount | Data persistence on host machine |

## 34.5.2 Daemon Process Configuration

For deployments not using Docker, MiniClaw needs to be configured as a system daemon.

### systemd (Linux)

```ini
# /etc/systemd/system/miniclaw.service

[Unit]
Description=MiniClaw AI Assistant
After=network.target

[Service]
Type=simple
User=miniclaw
WorkingDirectory=/opt/miniclaw
ExecStart=/usr/bin/node dist/index.js
Restart=on-failure
RestartSec=5s

# Environment variables
EnvironmentFile=/opt/miniclaw/.env

# Security hardening
NoNewPrivileges=true
ProtectSystem=strict
ProtectHome=true
ReadWritePaths=/opt/miniclaw/.miniclaw

# Resource limits
MemoryMax=512M
CPUQuota=100%

[Install]
WantedBy=multi-user.target
```

```bash
# Install service
sudo systemctl daemon-reload
sudo systemctl enable miniclaw
sudo systemctl start miniclaw

# Check status
sudo systemctl status miniclaw
sudo journalctl -u miniclaw -f
```

## 34.5.3 Security Hardening

Referencing OpenClaw's security model (Chapter 24), MiniClaw's minimum security checklist:

### 1. Authentication Protection

```typescript
// Add simple token authentication to Gateway

const GATEWAY_TOKEN = process.env.MINICLAW_TOKEN;

gateway.addMiddleware((client, frame) => {
  if (!GATEWAY_TOKEN) return true;  // Skip if not configured
  if (client.authenticated) return true;

  if (frame.type === "req" && frame.method === "auth") {
    const token = (frame.params as { token: string })?.token;
    if (token === GATEWAY_TOKEN) {
      client.authenticated = true;
      return true;
    }
    return false;  // Authentication failed
  }
  return false;  // Unauthenticated requests rejected
});
```

### 2. Tool Safety

```typescript
// Enhanced Bash tool security

const BLOCKED_COMMANDS = [
  /rm\s+-rf\s+\//,
  /mkfs/,
  /dd\s+if=\/dev/,
  />(\/etc|\/var|\/usr)/,
  /curl.*\|\s*(bash|sh)/,
  /wget.*\|\s*(bash|sh)/,
];

const ALLOWED_PATHS = [
  process.cwd(),
  path.join(os.homedir(), ".miniclaw"),
];

function isSafeCommand(command: string): boolean {
  for (const pattern of BLOCKED_COMMANDS) {
    if (pattern.test(command)) return false;
  }
  return true;
}

function isSafePath(filePath: string): boolean {
  const resolved = path.resolve(filePath);
  return ALLOWED_PATHS.some(allowed => resolved.startsWith(allowed));
}
```

### 3. Input Sanitization

```typescript
// Prevent prompt injection

function sanitizeInput(message: string): string {
  // Remove potential system prompt injection attempts
  const dangerous = [
    /ignore (previous|above|all) instructions/i,
    /you are now/i,
    /system:\s/i,
    /\[SYSTEM\]/i,
  ];

  let sanitized = message;
  for (const pattern of dangerous) {
    if (pattern.test(sanitized)) {
      sanitized = `[User message - potentially contains instruction override attempt]\n\n${sanitized}`;
      break;
    }
  }
  return sanitized;
}
```

### 4. Reverse Proxy

In production, it's recommended to use Caddy or Nginx as a reverse proxy:

```
# Caddyfile

miniclaw.example.com {
    # WebSocket proxy
    @ws {
        header Connection *Upgrade*
        header Upgrade websocket
    }
    reverse_proxy @ws localhost:3000

    # WebChat UI
    reverse_proxy /chat/* localhost:8080

    # Automatic HTTPS
    tls {
        protocols tls1.3
    }

    # Rate limiting
    rate_limit {
        zone dynamic {
            key {remote_host}
            events 60
            window 1m
        }
    }
}
```

### Security Checklist

| Item | Status | Description |
|------|--------|-------------|
| Gateway Token Authentication | Required | Prevent unauthorized access |
| HTTPS / WSS | Required | Encrypt transmission |
| Non-root Execution | Required | Least privilege |
| Tool Command Whitelist | Recommended | Restrict dangerous operations |
| File Path Restriction | Recommended | Limit to working directory |
| Input Sanitization | Recommended | Prevent prompt injection |
| Rate Limiting | Recommended | Prevent abuse |
| Audit Logging | Optional | Log all tool invocations |
| Automatic Backup | Optional | Regularly backup session and memory data |
