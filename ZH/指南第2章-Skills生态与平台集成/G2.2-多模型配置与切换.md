# G2.2 多模型配置与切换
> **生成模型**：Claude Opus 4.6 (anthropic/claude-opus-4-6)
> **Token 消耗**：输入 ~180k tokens，输出 ~7k tokens（本节）

---

如果说 Skills 决定“AI 会做什么”，那模型配置决定的就是“AI 用谁来做”。OpenClaw 在这件事上做得很实用：你既可以只配一个默认模型，也可以同时接多个提供者，再加上回退链，做成比较稳的生产配置。

## G2.2.1 默认模型从哪里来

第一次安装时，`openclaw onboard` 会引导你选一个默认模型。这个设置不是一次性的，后面随时能改。

直接看当前状态：

```bash
openclaw models
openclaw models status
openclaw models list --all
```

OpenClaw 当前支持的模型提供者很多，常见的有 **Anthropic**（Claude）、**OpenAI**（GPT）、**Google**（Gemini）、**Mistral**，以及一大批兼容 OpenAI 或 Anthropic 协议的第三方平台。

## G2.2.2 API Key 放哪儿

最常见有两种方式：环境变量，或者 `openclaw.json` 里的 `models.providers`。

先看环境变量写法：

```bash
export OPENAI_API_KEY="sk-..."
export ANTHROPIC_API_KEY="sk-ant-..."
export GEMINI_API_KEY="AIza..."
```

如果你更喜欢集中配置，可以写到 `openclaw.json`：

```json5
{
  agents: {
    defaults: {
      model: { primary: "openai/gpt-4o" },
    },
  },
  models: {
    providers: {
      openai: {
        apiKey: "$OPENAI_API_KEY",
      },
      anthropic: {
        apiKey: "$ANTHROPIC_API_KEY",
      },
    },
  },
}
```

> **提示**：书写时建议继续引用环境变量，而不是把真实密钥直接硬编码进配置文件。这样更适合多人协作，也更安全。

## G2.2.3 怎么切换模型

CLI 里最直接：

```bash
openclaw models set openai/gpt-4o
openclaw models set anthropic/claude-sonnet-4-6
openclaw models set-image google/gemini-2.5-flash-image
```

在聊天里也能切：

```text
/model
/model list
/model openai/gpt-4o
/model anthropic/claude-sonnet-4-6
/model status
```

这里的 **会话命令**（Slash Command）本质上是对当前会话的运行时覆盖，不一定会改掉全局默认配置。

## G2.2.4 给模型配回退链

回退链（Fallback Chain）很重要。它解决的是“主模型偶尔掉线、限流、余额不足、上下文不兼容”这些现实问题。

你可以用命令维护：

```bash
openclaw models fallbacks list
openclaw models fallbacks add anthropic/claude-sonnet-4-6
openclaw models fallbacks add google/gemini-2.5-pro
openclaw models fallbacks remove google/gemini-2.5-pro
```

也可以直接写配置：

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "openai/gpt-4o",
        fallbacks: [
          "anthropic/claude-sonnet-4-6",
          "google/gemini-2.5-pro"
        ],
      },
    },
  },
}
```

> **衍生解释**：这里的“回退”不是负载均衡。它不是把请求平均分给多个模型，而是主模型失败后，按顺序尝试后备模型。

## G2.2.5 免费模型扫描：OpenRouter

如果你想找可白嫖或成本更低的模型，可以让 OpenClaw 扫描 OpenRouter 的免费目录：

```bash
openclaw models scan
openclaw models scan --no-probe
openclaw models scan --set-default
```

这一步特别适合新手快速试水，也适合在 VPS 上先搭一个“能跑就行”的默认配置。

## G2.2.6 中国模型怎么接入

对中文用户来说，DeepSeek、通义千问（Qwen）、月之暗面（Moonshot/Kimi）、智谱（Zhipu/GLM）往往更便宜，延迟也更友好。OpenClaw 的思路不是给每家都做一套特殊逻辑，而是尽量通过 **自定义提供者**（Custom Provider）统一接入。

以 DeepSeek 为例，可以直接接 OpenAI 兼容端点：

```json5
{
  agents: {
    defaults: {
      model: { primary: "deepseek/deepseek-chat" },
    },
  },
  models: {
    mode: "merge",
    providers: {
      deepseek: {
        baseUrl: "https://api.deepseek.com",
        apiKey: "$DEEPSEEK_API_KEY",
        api: "openai-completions",
        models: [
          {
            id: "deepseek-chat",
            name: "DeepSeek Chat",
            reasoning: false,
            input: ["text"],
            contextWindow: 64000,
            maxTokens: 8192,
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
          },
        ],
      },
    },
  },
}
```

Moonshot、智谱、本地代理、企业网关，接法也类似：核心就是 `baseUrl + apiKey + api + models[]`。

## G2.2.7 OAuth 与 Auth Profile

并不是所有模型都只吃 API Key。比如 Claude Pro/Max 这类订阅型入口，OpenClaw 可以走 OAuth 或对应 CLI 认证流程；而多数国产平台、OpenRouter、本地代理，还是 API Key 更常见。

你可以用这些命令排查认证状态：

```bash
openclaw models auth add
openclaw models auth login --provider qwen-portal --set-default
openclaw models status --probe
```

Auth Profile 可以理解成“认证档案”。同一个提供者可以有多个账号、多个 token，再由系统决定当前用哪一个。

## G2.2.8 模型别名

模型 ID 往往又长又难记，所以别名很有用。当前版本里用的是复数命令 `aliases`：

```bash
openclaw models aliases list
openclaw models aliases add gpt openai/gpt-4o
openclaw models aliases add ds deepseek/deepseek-chat
openclaw models aliases remove ds
```

配置完成后，你在聊天里就能直接：

```text
/model gpt
/model ds
```

## G2.2.9 一个实用配置模板

下面是一份比较接地气的多模型配置：

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "deepseek/deepseek-chat",
        fallbacks: ["openai/gpt-4o", "anthropic/claude-sonnet-4-6"],
      },
    },
  },
  models: {
    providers: {
      deepseek: {
        baseUrl: "https://api.deepseek.com",
        apiKey: "$DEEPSEEK_API_KEY",
        api: "openai-completions",
        models: [{ id: "deepseek-chat", name: "DeepSeek Chat" }],
      },
    },
  },
}
```

这个思路很适合国内用户：主模型走低延迟、低成本；回退链挂国际大模型保底。

## 本节小结

OpenClaw 的模型层并不复杂：先选一个默认模型，再配认证，再按需要加回退链、别名和自定义提供者。对中国用户来说，最有实战价值的是学会通过 `models.providers` 接 DeepSeek、Qwen、Moonshot、智谱这类平台。模型提供者的详细实现见第 8 章。
