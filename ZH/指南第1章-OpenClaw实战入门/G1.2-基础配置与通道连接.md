# G1.2 基础配置与通道连接

> **生成模型**：Claude Opus 4.6 (anthropic/claude-opus-4-6)
> **Token 消耗**：输入 ~180k tokens，输出 ~7k tokens（本节）

---

前一节我们已经把 OpenClaw 跑起来了，但它还只是“在本机终端里可以对话”。这一节我们把它接到真正的聊天通道里。

## G1.2.1 配置文件在哪里

OpenClaw 默认把主配置文件放在：

```bash
~/.openclaw/openclaw.json
```

虽然文件名叫 `.json`，但它使用的是 **JSON5**（JSON5）格式，可以写注释和尾随逗号。

你可以先确认目录存在：`ls ~/.openclaw`

> **衍生解释 — JSON5**
>
> JSON5 可以理解成“更适合人类手写的 JSON”。它允许注释和尾随逗号，所以维护配置时没那么容易被语法细节绊住。

## G1.2.2 先看懂配置骨架

你不用一开始就把全部字段背下来，先抓住四个核心部分：`gateway`、`agents`、`models`、`channels`。

```json5
{
  gateway: {
    port: 18789,
  },
  agents: {
    defaults: {
      model: "anthropic/claude-sonnet-4-5",
    },
  },
  models: {
    default: "anthropic/claude-sonnet-4-5",
  },
  channels: {
    telegram: {
      enabled: false,
    },
    discord: {
      enabled: false,
    },
  },
}
```

它们分别负责 Gateway 行为、Agent 默认值、模型认证和消息通道。

如果你现在只想先连一个通道，优先看 `channels` 就够了。

## G1.2.3 先接 Telegram：最容易上手

对大多数初学者来说，Telegram 是最省事的第一站。你先在 Telegram 里找到 `@BotFather`，创建机器人并拿到 bot token。

步骤很简单：

1. 在 Telegram 搜索 `@BotFather`
2. 发送 `/newbot`
3. 按提示给机器人起名字
4. 复制生成的 token

然后在终端里配置：

```bash
openclaw channels add --channel telegram --token "123456:ABCDEF_your_bot_token"
openclaw channels status --probe
```

这里要特别说明一下：你可能会直觉地写成“Telegram login”，但当前 OpenClaw CLI 的实际命令是 `channels add`。`channels login` 更常见于 WhatsApp 这类扫码配对通道。

也可以直接写进配置文件：

```json5
{
  channels: {
    telegram: {
      enabled: true,
      botToken: "123456:ABCDEF_your_bot_token",
      dmPolicy: "pairing",
    },
  },
}
```

配好以后，启动或重启 Gateway：

```bash
openclaw gateway
```

然后在 Telegram 里给机器人发一条消息，比如“你能帮我列一个明天的学习计划吗？”。如果是首次私聊，按配对提示完成授权就行。

## G1.2.4 再接 Discord：适合做群聊和工作流

Discord 的准备工作比 Telegram 多一点：创建应用、添加 Bot、复制 token、开启必要的 Intent。

建议步骤：

```bash
# 先把 token 写入 OpenClaw 配置
openclaw channels add --channel discord --token "YOUR_DISCORD_BOT_TOKEN"

# 检查通道状态和权限探测
openclaw channels status --probe
openclaw channels capabilities --channel discord
```

对应的配置大概长这样：

```json5
{
  channels: {
    discord: {
      enabled: true,
      token: "YOUR_DISCORD_BOT_TOKEN",
      commands: {
        native: true,
      },
    },
  },
}
```

在 Discord 端，至少确认：

- Bot 已被邀请进服务器
- 已开启 `Message Content Intent`
- Bot 具备读消息、发消息、读历史记录权限

最小验证方式就是直接私信这个 Bot。

## G1.2.5 通道接通后怎么验证

验证思路很朴素：你从外部通道发一条消息，看 OpenClaw 能不能回。

```typescript
const channelFlow = ["通道消息", "Gateway", "Agent 会话", "AI 模型", "原通道回复"];
```

你可以按这个顺序检查：

```bash
openclaw channels list
openclaw channels status --probe
openclaw gateway status
```

然后在 Telegram 或 Discord 里发送一句具体的话，例如：

- “请用三条 bullet 总结今天的待办”
- “帮我解释一下什么是 WebSocket”

如果 AI 能在对应聊天窗口里回你，说明链路已经打通。

> **提示**
>
> 调试通道问题时，不要一上来就怀疑模型。大多数“没回复”问题，实际出在 token、权限、Intent、配对策略或者 Gateway 没启动。

## G1.2.6 其他通道先知道名字就行

除了 Telegram 和 Discord，OpenClaw 还支持：

- WhatsApp（基于 **Baileys**（Baileys））
- Slack
- Signal
- iMessage

它们的接入方式各不相同：有的靠 bot token，有的要扫码登录，有的还依赖宿主系统或额外服务。所以学习顺序上，先把 Telegram 或 Discord 跑通最划算。通道适配器的详细实现见第 11-13 章。

## 本节小结

1. OpenClaw 的主配置文件在 `~/.openclaw/openclaw.json`，格式是 JSON5。
2. 初学时先抓住 `gateway`、`agents`、`models`、`channels` 这四块配置骨架。
3. Telegram 是最容易接入的通道，实际 CLI 命令是 `openclaw channels add --channel telegram --token ...`。
4. Discord 需要 bot token、Intent 和权限配置，接好后建议用 `channels status --probe` 做检查。
5. 通道消息的本质路径仍然是“聊天平台 -> Gateway -> Agent -> 聊天平台”，后面的源码章节会把这条路径彻底拆开。
